# MiAirX 是什么？

---

## 一句话版本

> 一个软件，装在你电脑上，让你的小米智能音箱被 QQ 音乐发现和投送。

---

## 说人话

**你现在的痛点**：买了小爱音箱，想在 QQ 音乐里点"投屏到音箱"，但列表里看不到你的小爱音箱。

**为什么**：因为小爱音箱说的是"小米语"，QQ 音乐说的是"行业通用语"（DLNA），它俩互相听不懂。

**MiAirX 干什么**：在你电脑上跑一个翻译官，把"行业通用语"翻译成"小米语"，于是 QQ 音乐就认识你的小爱音箱了。

---

## 打个比方

```
你出国旅游，不会当地语言，找了个翻译：

    你         翻译         当地人
  (说中文)  →  (翻译)  →  (说外语)
  
MiAirX 就是做这个翻译：

    QQ音乐        MiAirX        小爱音箱
  (说DLNA)  →  (翻译)  →  (说MiNA API)
```

---

## 它怎么工作的？（超简版）

```
第一步：QQ音乐说"附近有音箱吗？"
    ↓
第二步：MiAirX举手"有！这里有个小爱音箱Play增强版"
    ↓
第三步：QQ音乐说"好，这首歌给你，从这里开始播"
    ↓
第四步：MiAirX把歌转交给小爱音箱
    ↓
第五步：音箱响了 🎵
```

# 什么是 DLNA？

---

## 一句话版本

> DLNA 就是一个"行业标准"，让音乐 App 和音箱能互相认识和控制。

---

## 日常生活类比

你家的 Wi-Fi 就是一个局域网。在同一个 Wi-Fi 下：

| 你想做的事 | DLNA 的工作 |
|-----------|------------|
| 手机找音箱 | DLNA 说："大家好，我是音箱，我能放歌" |
| 手机上点播放 | DLNA 说："音箱，播这首歌！" |
| 手机看进度条 | DLNA 说："音箱，你播到几秒了？" |
| 音箱状态变了 | DLNA 说："手机，我暂停了哦" |

DLNA 就是这套"打招呼 + 下命令 + 问状态 + 通知变化"的整套规矩。

---

## 三个角色（记三个字母就行）

| 角色 | 你是谁 | 做什么 |
|------|--------|--------|
| **DMR** | 音箱 | 负责"播放" |
| **DMC** | 手机上的音乐 App | 负责"控制" |
| **DMS** | 存歌的地方 | 负责"提供音乐" |

MiAirX 做的事：**假装自己是一个标准音箱（DMR）**，QQ 音乐以为自己在跟一个标准音箱说话，实际上 MiAirX 在背后翻译给小米音箱。

---

## DLNA 发现设备的过程

```
你打开QQ音乐投屏列表的那一刻：

1. QQ音乐大喊（对整个Wi-Fi广播）：
   "谁是音箱？谁是音箱？"

2. MiAirX举手：
   "我是！我叫'小爱音箱Play增强版'，这是我的地址"

3. QQ音乐去访问地址：
   "你会做什么？"

4. MiAirX递上名片（设备描述）：
   "我会：播放、暂停、跳转、调音量、报告进度"

5. QQ音乐说：
   "好，以后你状态有变化记得通知我"

6. MiAirX说：
   "没问题，这是我的订阅号"

--- 至此，连接建立 ---
```

# 完整流程：一首歌的旅程

---

## 从点击到出声

```
时间点   发生了什么

00:00    你在QQ音乐点了一首歌，点"投屏到小爱音箱"
         ↓
00:01    QQ音乐给MiAirX发第一条消息：
         "这首歌的链接是 https://qqmusic.xxx/song.mp3"
         ↓
00:01    MiAirX做了两件事：
         ① 自己默默去下载这首歌（缓冲到内存）
         ② 生成一个新链接：http://你的电脑IP:8200/media/随机码
         ③ 告诉音箱："播这个！http://你的电脑IP:8200/media/随机码"
         ↓
00:02    音箱开始从MiAirX拉歌（从你电脑直接传，不走外网）
         ↓
00:02    MiAirX告诉QQ音乐："已经在播了！"
         ↓
00:03    你的耳朵：🎵🎶🎵
```

---

## 进度条是怎么动的

```
MiAirX记住两个数字：
  - 已经播了多少秒（比如50秒）
  - 这一轮是从什么时候开始播的（比如16:30:00）

QQ音乐每秒问一次："播到哪了？"

MiAirX心算：
  16:30:00 到现在过了3秒
  50 + 3 = 53秒
  "播到53秒了！"

QQ音乐进度条显示：0:53
```

---

## 你为什么能拖进度条

```
你把进度条拖到 2:30 → QQ音乐说："跳转到150秒处"
                           ↓
MiAirX做三件事：         ① 停止音箱当前播放
                         ② 重新下载，从第150秒的地方开始
                         ③ 告诉音箱："从头播这个新链接"
                         ④ 自己记住："累积时间=150秒，开始计时"
                           ↓
                         音箱从150秒处继续播放
```

# 核心魔法：用了三个"分身"

---

## 分身1：翻译官

```
MiAirX接到的命令 → 翻译 → 发给小米云

"播放"            →  play_url("http://...")  →  小米云推给音箱
"暂停"            →  player_pause()          →  小米云推给音箱  
"音量50"          →  player_set_volume(50)   →  小米云推给音箱
"你什么状态？"    →  player_get_status()     →  小米云查音箱
```

---

## 分身2：快递员

```
为什么需要快递员？

QQ音乐给的链接：https://qqmusic.xxx/stream?token=超长参数&expires=...
直接给音箱用 → 可能太长、格式不对、或者过期了

MiAirX的做法：
① 自己先下载到内存（快递员去取件）
② 用电脑当本地服务器（快递员放到你家门口）
③ 让音箱来电脑上取（音箱自己走几步就拿到了）

音箱拿到的链接变成：http://192.168.1.172:8200/media/简短的随机码

短、快、稳定，还支持拖进度条。
```

---

## 分身3：状态影子

```
为什么需要一个"影子"？

因为小米说不清楚音箱在干嘛：
  "音箱在播吗？"  →  小米API："没在播"  ← 但其实在播！它不认得代理流！

所以 MiAirX 自己记：
  ✅ 什么时候开始播的
  ✅ 已经播了多久  
  ✅ 是不是用户自己暂停的

翻译官按小米说的汇报 → 不准
影子自己记录的位置    → 准的

MiAirX 用影子来报告进度。
```

# 最难的坑：进度条总往回跳

---

## 现象

```
正常播放一首歌：
  进度条：0:50 → 0:51 → 0:52 → 0:50 ← 咦？怎么跳回去了！
```

## 为什么会这样

```
① 小米API说："音箱没在播"（但其实在播！它认不出代理流）
           ↓
② MiAirX以为音箱停了，状态变成"暂停"
           ↓
③ 进度计时停止——进度条卡住不动了
           ↓
④ QQ音乐发现："嗯？进度条卡住不动了？出问题了！"
           ↓
⑤ QQ音乐主动发"播放"命令试图修复
           ↓
⑥ 但此时 MiAirX 记录的位置是卡住之前的旧时间
           ↓
⑦ 用旧时间重新开始 → 进度条跳回去了！
```

## 怎么修的

```
关键是让 MiAirX 明白：代理模式下，小米说"没在播"是骗人的。

修好之后：
  小米API说"没在播" → MiAirX说"你骗人，我当没听见"
  进度条继续正常走  ✓

这就是修了好几天的核心问题。
```

# 这套东西怎么用

---

## 你需要准备

- [ ] 一台电脑（Windows/Mac/Linux 都行）
- [ ] 一个小米智能音箱（在同一个 Wi-Fi 下）
- [x] 不需要在音箱上装任何东西
- [x] 不需要改动音箱固件
- [x] 不需要 root 音箱

## 三步走

```
第一步：填配置（小米账号、音箱ID、电脑IP）
         ↓
第二步：双击 start.bat（Windows）或运行 python start.py
         ↓
第三步：打开QQ音乐 → 投屏列表 → 看到"小爱音箱Play增强版"→ 投！
```
